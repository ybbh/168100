# MuduDB 早期开发贡献奖励计划

**版本：V1.0**  
**项目方：MuduDB 项目组**  
**发布日期：2026年5月8日**  
**联系方式：168100 [at] scupt [dot] com**

## 一、计划说明

MuduDB 早期开发贡献奖励计划旨在鼓励开发者基于真实、合法、可核验的软件开发贡献，参与 MuduDB 项目建设。

本计划仅适用于与 MuduDB 软件开发直接相关的贡献，包括代码开发、测试用例、Bug 修复、性能优化、构建工具、开发工具链、必要技术文档，以及其他经项目方确认的开发工作。

运营、市场、商务、宣传、活动组织、非技术文案等非开发工作，不适用本计划，除非项目方另行书面确认。

本计划及相关 SN 登记信息仅用于 MuduDB 早期开发贡献的内部记录、贡献纪念、登记核验和项目管理。除经参与者主动提交或同意公开的脱敏信息外，项目方不公开参与者个人敏感信息。

本计划不构成公开招募、融资、集资、投资、理财、代币发行、彩票、抽奖或任何形式的收益承诺。

如希望参与 MuduDB 早期开发贡献计划，请先联系项目方确认参与资格、开发任务、登记方式及适用条款。

## 二、非融资、非投资声明

本计划不构成融资、集资、投资、理财、借贷、证券发行、股权发行、代币发行、彩票、抽奖或赌博活动。

参与者无需向项目方支付任何费用。项目方不承诺本金返还、固定收益、保底收益、投资回报或收益分成。

本计划项下奖励仅基于参与者对 MuduDB 项目的实际开发贡献、项目方确认结果及项目里程碑情况产生。

## 三、开发任务与贡献确认

项目方可通过 Issue、Pull Request、任务单、邮件、即时通讯记录、代码仓库记录、在线系统或补充协议等方式发布、确认或验收开发任务。

参与者完成开发任务后，项目方根据代码质量、功能完成度、测试结果、性能指标、代码审查结果、合并状态和项目实际需要，确认是否构成有效开发贡献。

未经项目方确认的代码提交、建议、讨论或实验性实现，不当然产生奖励资格。

## 四、奖励登记与发放

对经确认的有效开发贡献，项目方可建立奖励登记记录。奖励登记记录可包括任务内容、代码仓库记录、Commit / PR / Issue 编号、验收结果、奖励基准金额、发放条件、发放时间和奖励上限等信息。

项目达到约定里程碑，且项目方依法确认奖励池可启动、相关资金足以覆盖拟发放奖励及相关税费后，项目方可根据奖励登记记录发放 MuduDB 项目成功奖励。

相关里程碑包括但不限于：

1. MuduDB 获得实际商业收入、外部合作、采购、资助、补助或其他合法项目资金，且相关资金实际到账；
2. 项目方在综合项目收入、到账资金、运营成本、税费负担和奖励登记情况后，依法确认可启动奖励池。

如项目未达到约定里程碑，或奖励池未依法启动，项目方无义务发放项目成功奖励；但不影响参与者已经依法取得、双方已明确约定且不以项目成功为条件的其他开发报酬。

## 五、SN 纪念编号

项目方可为经确认的 MuduDB 早期开发贡献生成或登记 SN 纪念编号。SN 纪念编号的来源可以包括系统生成编号、实物纪念编号或项目方确认的其他合法编号来源。

SN 纪念编号仅用于贡献纪念、登记核验和内部管理，不构成兑付凭证、债权凭证、投资凭证、支付凭证、代币、彩票、抽奖凭证、收益权凭证或可转让权益。

奖励资格以项目方奖励登记记录、开发任务记录、真实代码贡献记录和身份核验结果为准，不以持有任何实物或知悉任何 SN 纪念编号作为唯一依据。

参与者不得买卖、转让、炒作、质押、包装、宣传 SN 纪念编号或其他纪念物；不得将 SN 纪念编号解释、宣传或使用为投资产品、金融产品、兑付凭证、代币、收益权或可交易权益。

## 六、公开信息与隐私保护

在参与者按照登记规则主动提交公开登记信息，或另行同意公开的前提下，项目方可公开展示部分奖励登记信息，以提高计划透明度和公信力。

公开信息应限于项目名称、任务编号、贡献类型、代码仓库记录、验收状态、SN 纪念编号脱敏信息、奖励状态等非敏感信息。

项目方公开展示奖励登记信息时，应对 SN 纪念编号及个人信息进行必要脱敏处理，不公开参与者身份证件、联系方式、收款账户、完整 SN、税务资料等敏感信息。
