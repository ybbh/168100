# MuduDB 贡献纪念 SN 登记流程说明

**版本：V1.0**  
**适用对象：MuduDB 早期开发贡献参与者**  
**项目方指定邮箱：168100 [at] scupt [dot] com**  
**生效日期：2026年5月8日**

## 一、适用范围

本流程适用于参与者完成经项目方确认的 MuduDB 早期开发贡献后，进行 SN 纪念编号登记、兑付记录登记及公开信息提交。

本流程中的 SN 纪念编号仅用于贡献纪念、登记核验和内部管理，不构成兑付凭证、债权凭证、投资凭证、支付凭证、代币、彩票、抽奖凭证、收益权凭证或可转让权益。

奖励资格、奖励金额、兑付条件和兑付状态，以项目方保存的完整登记记录、真实开发贡献记录、代码仓库记录、身份核验结果及双方确认的正式条款为准。

## 二、登记方式总览

参与者完成经项目方确认的 MuduDB 早期开发贡献后，应按以下顺序完成登记：

1. 向项目方指定邮箱发送个人完整 SN 登记邮件；
2. 在项目方指定的公开代码仓库中，通过 Pull Request 提交对应的公开登记 CSV 文件。

邮件用于保存完整登记信息；公开仓库 Pull Request 用于提交脱敏后的公开登记信息。

## 三、个人完整登记邮件

参与者应将个人完整登记信息发送至项目方指定邮箱：

```text
__________
```

### 1. 邮件标题格式

邮件标题应使用以下格式：

```text
SN 登记 - 参与者姓名
```

示例：

```text
SN 登记 - 张三
```

### 2. 邮件正文格式

邮件正文应严格使用以下格式：

```text
参与者姓名：
参与者唯一名：
联系方式：
邮箱：
完整 SN：
面值：
```

示例：

```text
参与者姓名：张三
参与者唯一名：zhangsan
联系方式：13800000000
邮箱：zhangsan@example.com
完整 SN：AA12345678
面值：10
```

如项目方因核验需要要求补充对应任务编号、收款信息、税务信息或其他必要材料，参与者应予以配合。

### 3. 邮件信息用途

邮件中的完整 SN 信息及个人信息仅用于以下目的：

1. 身份核验；
2. 贡献登记；
3. 奖励发放；
4. 税费处理；
5. 争议解决；
6. 项目方内部登记和留档。

项目方公开展示登记信息时，不公开参与者身份证件、联系方式、收款账户、完整 SN、税务资料或其他敏感个人信息。

## 四、公开登记信息 Pull Request

参与者应在项目方指定的公开代码仓库中，通过 Pull Request 提交或更新对应的公开登记文件。

### 1. 文件命名格式

公开登记文件命名格式为：

```text
参与者唯一名.csv
```

其中，“参与者唯一名”应使用项目方认可的唯一标识，例如：

1. GitHub 用户名；
2. 开发者 ID；
3. 项目方认可的其他公开唯一名称。

参与者唯一名不得包含身份证件号码、手机号、住址、完整收款账户等敏感个人信息。

示例：

```text
participants/zhangsan.csv
```

## 五、公开 CSV 格式

公开 CSV 文件建议使用 UTF-8 编码。除表头外，每一行表示一次登记、兑付、更正或取消操作。

公开 CSV 文件应使用以下字段：

```csv
SN,面值,操作,时间,备注
```

字段说明如下：

| 字段 | 说明 |
|---|---|
| SN | 脱敏后的 SN 纪念编号，不得填写完整 SN |
| 面值 | SN对应的面值 |
| 操作 | 填写“登记”“兑付”“更正”或“取消” |
| 时间 | 操作发生时间，建议使用 `YYYY-MM-DD HH:MM:SS` 格式 |
| 备注 | 可填写任务编号、说明信息或其他非敏感备注 |

其中，“面值”仅用于记录对应纪念物的信息，不代表奖励金额、兑付金额、奖励、收益计算依据或任何兑付依据。
“兑付”仅指项目方根据正式条款、奖励登记记录、真实开发贡献记录、身份核验结果、税费处理结果及实际支付情况，完成项目成功奖励发放；不表示 SN 纪念编号本身具有兑付凭证、债权凭证、投资凭证、支付凭证、收益权凭证或可转让权益属性。
公开 CSV 仅记录兑付状态，不公开实际兑付金额、收款账户、税务资料或其他个人敏感信息。

### 1. 示例 CSV

```csv
SN,面值,操作,时间,备注
AA****5678,10,登记,2026-05-08 12:00:00,完成首次登记
AA****5678,10,兑付,2026-05-09 15:30:00,项目成功奖励已完成发放
```

### 2. SN 脱敏规则

公开 CSV 中不得填写完整 SN。建议使用以下脱敏格式：

```text
前两位 + **** + 后四位
```

示例：

```text
AA****5678
```

如 SN 格式不适合上述脱敏方式，参与者应采用足以避免还原完整 SN 的脱敏方式，并确保公开信息不包含敏感个人信息。

## 六、只追加不删除原则

公开 CSV 文件中的记录用于公开展示和外部核验。为保证登记过程可追溯，CSV 文件中的记录原则上：

1. 不删除历史记录；
2. 不覆盖历史记录；
3. 不通过修改旧行掩盖历史变更；
4. 后续变更应通过新增行的方式追加记录；
5. 如需更正明显错误，应追加“更正”记录并在备注中说明原因。

示例：

```csv
SN,面值,操作,时间,备注
WR****5789,10,登记,2026-05-08 12:00:00,登记备注
WR****5789,10,更正,2026-05-08 13:00:00,更正第X行登记备注
WR****5789,10,兑付,2028-05-09 15:30:00,奖励已完成发放
```

如公开仓库中误提交完整 SN、联系方式、身份证件号码、收款账户、税务资料等敏感信息，项目方可基于隐私保护、安全和合规需要，要求修改、关闭 PR、删除相关内容或在必要时采取仓库历史清理措施。

## 七、登记、兑付、更正与取消操作说明

### 1. 登记

“登记”表示参与者已向项目方提交完整 SN 登记邮件，并在公开仓库中提交对应的脱敏公开登记信息。

示例：

```csv
SN,面值,操作,时间,备注
WR****5789,10,登记,2026-05-08 12:00:00,完成首次登记
```

### 2. 兑付

“兑付”表示项目方已根据《MuduDB 早期开发贡献奖励计划》、项目方保存的完整登记记录、真实开发贡献记录、身份核验结果、税费处理结果及实际支付情况，完成对应项目成功奖励发放。

“兑付”记录仅用于公开展示奖励发放状态，不表示 SN 纪念编号本身构成兑付凭证、债权凭证、投资凭证、支付凭证、收益权凭证或可转让权益。

示例：

```csv
SN,面值,操作,时间,备注
WR****5789,10,兑付,2028-05-09 15:30:00,项目成功奖励已完成发放
```

### 3. 更正

“更正”用于修正前序记录中的明显错误。更正记录应在备注中说明被更正事项及原因、被更正记录行（行号从1行开始算，表头也算一行）。不得通过删除或覆盖旧记录的方式掩盖历史变更。

示例：

```csv
SN,面值,操作,时间,备注
WR****5789,10,更正,2026-05-08 13:00:00,更正第X行登记备注
```

### 4. 取消

“取消”用于记录项目方确认该 SN 登记不再有效、重复登记、登记错误或其他需要终止公开登记状态的情况。取消记录应在备注中说明被取消事项及其原因、被取消记录行（行号从1行开始算，表头也算一行）

示例：

```csv
SN,面值,操作,时间,备注
WR****5789,10,取消,2026-05-08 14:00:00,重复登记，取消第X行公开登记
```

## 八、信息一致性要求

参与者通过邮件提交的完整 SN，应与公开 CSV 文件中的脱敏 SN 对应。

公开 CSV 仅用于透明展示和外部核验，不作为奖励资格、奖励金额、兑付条件或兑付状态的唯一依据。

项目方有权对邮件登记信息、公开 CSV 信息、代码仓库记录、任务验收记录和身份核验结果进行一致性核验。如信息不一致，项目方可要求参与者补充、更正或重新提交。

## 九、公开信息禁止内容

公开 CSV 文件、Pull Request 标题、Pull Request 正文、评论区及其他公开提交内容中不得包含以下内容：

1. 参与者身份证件号码；
2. 手机号；
3. 住址；
4. 完整收款账户；
5. 完整 SN；
6. 税务资料；
7. 未公开商业信息；
8. 未公开源代码或技术秘密；
9. 其他可能泄露个人隐私、项目秘密或第三方权益的信息；
10. 其他可能导致完整 SN 被还原、纪念物被炒作或登记信息被误解为可交易权益的信息。

如 Pull Request 中包含上述信息，项目方有权要求参与者修改、关闭 Pull Request、删除相关内容或采取必要的仓库历史清理措施。

## 十、Pull Request 说明建议格式

提交公开登记信息 Pull Request 时，建议使用以下说明格式：

```text
本 PR 用于提交 MuduDB 贡献纪念 SN 公开登记信息。

参与者唯一名：
SN：
面值：
本次操作：登记 / 兑付 / 更正 / 取消
说明：
```

示例：

```text
本 PR 用于提交 MuduDB 贡献纪念 SN 公开登记信息。

参与者唯一名：zhangsan
SN：WR****5789
面值：10
本次操作：登记
说明：完成首次 SN 脱敏登记
```

## 十一、项目方审核

项目方可根据以下内容审核公开登记 Pull Request：

1. 文件名是否符合“参与者唯一名.csv”格式；
2. CSV 字段是否符合 `SN,面值,操作,时间,备注` 格式；
3. SN 是否已经脱敏；
4. 是否存在敏感个人信息；
5. 是否遵守只追加不删除原则；
6. 邮件完整登记信息是否已提交；
7. 公开登记信息是否与项目方内部登记记录相对应。

不符合要求的 Pull Request，项目方有权要求参与者修改、补充或重新提交。
